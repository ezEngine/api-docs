Detailed Description

ezCommandLineOption (and derived types) are used to define options that the application supports.

Command line options are created as global variables anywhere throughout the code, wherever they are needed. The point of using them over going through ezCommandLineUtils directly, is that the options can be listed automatically and thus an application can print all available options, when the user requests help.

Consequently, their main purpose is to make options discoverable and to document them in a consistent manner.

Additionally, classes like ezCommandLineOptionEnum add functionality that makes some options easier to setup.

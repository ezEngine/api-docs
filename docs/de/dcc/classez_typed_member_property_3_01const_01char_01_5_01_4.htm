Detailed Description

Specialization of ezTypedMemberProperty for const char*.

This works because ezTypedMemberProperty< typename ezTypeTraits<Type>::NonConstReferenceType > in ezAccessorProperty does not actually remove the constness of the type but of the pointer, so const char* is not affected.
